① Application Stage

Runs each frame on the CPU. Updates fish x/y position, velocity, wobble, tail phase, jellyfish drift and pulse, bubble respawning, and all scene-level logic. This is where game/simulation state lives before anything is drawn.

② Geometry Stage

Computes per-object transforms: fish orientation derived from velocity vector, jellyfish pulse deformation via sine, coral sway offset, and frustum culling — objects outside the canvas are skipped before rasterization begins.

③ Rasterization Stage

Converts geometry to pixels using painter's algorithm (back→front): background → coral → seabed → bubbles → jellyfish → fish. Toggle Wireframe Mode to expose the raw geometric primitives before fill/shading is applied.
